# 반응형 테스트 가이드

이 문서는 Instagram SNS 프로젝트의 반응형 디자인을 테스트하기 위한 가이드입니다.

## 브레이크포인트

### Mobile (< 768px)

- **너비**: 320px ~ 767px
- **레이아웃 특징**:
  - Header 표시 (높이 60px)
  - BottomNav 표시 (높이 50px)
  - Sidebar 숨김
  - 전체 너비 PostCard
  - 프로필 이미지: 90px

### Tablet (768px ~ 1024px)

- **너비**: 768px ~ 1023px
- **레이아웃 특징**:
  - Icon-only Sidebar 표시 (너비 72px)
  - Header/BottomNav 숨김
  - PostCard 최대 너비: 630px
  - 프로필 이미지: 150px

### Desktop (1024px+)

- **너비**: 1024px 이상
- **레이아웃 특징**:
  - Full Sidebar 표시 (너비 244px)
  - Header/BottomNav 숨김
  - PostCard 최대 너비: 630px
  - 프로필 이미지: 150px

## 주요 페이지별 반응형 체크리스트

### 1. 홈 피드 페이지 (`/`)

#### Mobile (< 768px)

- [ ] Header가 상단에 표시되는지 확인
- [ ] BottomNav가 하단에 표시되는지 확인
- [ ] PostCard가 전체 너비로 표시되는지 확인
- [ ] 이미지가 1:1 정사각형 비율로 표시되는지 확인
- [ ] 더블탭 좋아요 기능이 작동하는지 확인
- [ ] 스크롤이 부드럽게 작동하는지 확인

#### Tablet (768px ~ 1024px)

- [ ] Icon-only Sidebar가 왼쪽에 표시되는지 확인
- [ ] PostCard가 최대 630px 너비로 중앙 정렬되는지 확인
- [ ] 이미지가 1:1 정사각형 비율로 표시되는지 확인
- [ ] 무한 스크롤이 정상 작동하는지 확인

#### Desktop (1024px+)

- [ ] Full Sidebar가 왼쪽에 표시되는지 확인
- [ ] PostCard가 최대 630px 너비로 중앙 정렬되는지 확인
- [ ] 이미지가 1:1 정사각형 비율로 표시되는지 확인
- [ ] 호버 효과가 정상 작동하는지 확인

### 2. 프로필 페이지 (`/profile/[userId]`)

#### Mobile (< 768px)

- [ ] 프로필 이미지가 90px로 표시되는지 확인
- [ ] 프로필 정보가 세로로 배치되는지 확인
- [ ] 게시물 그리드가 3열로 표시되는지 확인
- [ ] 통계(게시물 수, 팔로워 수, 팔로잉 수)가 가로로 배치되는지 확인
- [ ] 팔로우 버튼이 정상 작동하는지 확인

#### Tablet (768px ~ 1024px)

- [ ] 프로필 이미지가 150px로 표시되는지 확인
- [ ] 프로필 정보가 가로로 배치되는지 확인
- [ ] 게시물 그리드가 3열로 표시되는지 확인
- [ ] 통계가 정상적으로 표시되는지 확인

#### Desktop (1024px+)

- [ ] 프로필 이미지가 150px로 표시되는지 확인
- [ ] 프로필 정보가 가로로 배치되는지 확인
- [ ] 게시물 그리드가 3열로 표시되는지 확인
- [ ] 호버 시 좋아요/댓글 수가 표시되는지 확인

### 3. 게시물 상세 페이지 (`/post/[postId]`)

#### Mobile (< 768px)

- [ ] 전체 페이지로 표시되는지 확인
- [ ] 이미지가 상단에 표시되는지 확인
- [ ] 댓글 영역이 하단에 표시되는지 확인
- [ ] 스크롤이 부드럽게 작동하는지 확인
- [ ] 댓글 작성 폼이 접근 가능한지 확인

#### Tablet (768px ~ 1024px)

- [ ] 모달로 표시되는지 확인
- [ ] 이미지가 50% 너비로 표시되는지 확인
- [ ] 댓글 영역이 50% 너비로 표시되는지 확인
- [ ] 댓글 목록이 스크롤 가능한지 확인

#### Desktop (1024px+)

- [ ] 모달로 표시되는지 확인
- [ ] 이미지가 50% 너비로 표시되는지 확인
- [ ] 댓글 영역이 50% 너비로 표시되는지 확인
- [ ] 모달 닫기 버튼이 작동하는지 확인

## 터치 인터랙션 테스트

### 더블탭 좋아요

- **위치**: 게시물 이미지 영역
- **동작**: 이미지 영역을 빠르게 두 번 탭하면 좋아요가 추가됨
- **테스트 항목**:
  - [ ] 더블탭 시 좋아요가 추가되는지 확인
  - [ ] 큰 하트 애니메이션이 표시되는지 확인 (fade in/out, 1초)
  - [ ] 이미 좋아요가 눌려있으면 무시되는지 확인

### 스와이프

- **위치**: 게시물 이미지 (향후 확장 가능)
- **동작**: 이미지 좌우 스와이프로 다음/이전 이미지 이동 (현재 미구현)

### 터치 스크롤

- **위치**: 댓글 목록, 게시물 피드
- **동작**: 손가락으로 위아래 스크롤
- **테스트 항목**:
  - [ ] 스크롤이 부드럽게 작동하는지 확인
  - [ ] 스크롤 중에 다른 동작이 방해되지 않는지 확인
  - [ ] 무한 스크롤이 정상 작동하는지 확인

### 버튼 터치

- **테스트 항목**:
  - [ ] 모든 버튼이 충분한 터치 영역(최소 44x44px)을 가지는지 확인
  - [ ] 버튼 터치 시 즉시 반응하는지 확인
  - [ ] 터치 피드백이 명확한지 확인

## 테스트 도구

### Chrome DevTools

1. **개발자 도구 열기**: `F12` 또는 `Ctrl+Shift+I` (Windows) / `Cmd+Option+I` (Mac)
2. **디바이스 모드 활성화**: `Ctrl+Shift+M` (Windows) / `Cmd+Shift+M` (Mac)
3. **디바이스 선택**:
   - iPhone 12 Pro (390x844)
   - iPad Air (820x1180)
   - Custom (직접 크기 설정)

### Firefox DevTools

1. **개발자 도구 열기**: `F12`
2. **반응형 디자인 모드**: `Ctrl+Shift+M` (Windows) / `Cmd+Shift+M` (Mac)

### 실제 디바이스 테스트

- **모바일**: 실제 스마트폰에서 테스트하는 것이 가장 정확함
- **태블릿**: 실제 태블릿에서 테스트하는 것이 가장 정확함

## 주요 테스트 시나리오

### 시나리오 1: 모바일에서 홈 피드 탐색

1. 모바일 뷰포트로 설정 (375px)
2. 홈 페이지 접속
3. PostCard가 전체 너비로 표시되는지 확인
4. 스크롤하여 게시물 목록 확인
5. 더블탭으로 좋아요 추가 테스트
6. 게시물 클릭하여 상세 페이지 이동 확인

### 시나리오 2: 태블릿에서 프로필 보기

1. 태블릿 뷰포트로 설정 (768px)
2. 프로필 페이지 접속
3. Icon-only Sidebar가 표시되는지 확인
4. 프로필 정보가 가로로 배치되는지 확인
5. 게시물 그리드가 3열로 표시되는지 확인
6. 게시물 클릭하여 상세 모달 열기 확인

### 시나리오 3: 데스크톱에서 게시물 작성

1. 데스크톱 뷰포트로 설정 (1280px)
2. Full Sidebar가 표시되는지 확인
3. "만들기" 버튼 클릭하여 게시물 작성 모달 열기
4. 이미지 업로드 테스트
5. 캡션 입력 테스트
6. 게시물 업로드 후 피드에 반영되는지 확인
